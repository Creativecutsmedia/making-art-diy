# 3.1b Product Edit Form — kontrakt

**Status:** Kontrakt — låses INDEN kode skrives. W3 step 3.1b, 2026-04-26.
**Formål:** Definér admin-v2 produkt-edit-form's integration med 3.1a `admin-products-write` endpoint, inkl. field-mapping, state-machine, error-handling, og test-strategi.
**Output:** Reference-doc. Ingen kode i denne PR.

**Læs først:** `docs/3.1a-admin-products-write-contract.md`, `admin-v2/src/hooks.jsx`, `admin-v2/src/page_product_edit.jsx`, `admin-v2/src/page_products.jsx`.

## TL;DR

3.1b er admin-v2 frontend produkt-edit-formular der kalder 3.1a `POST /.netlify/functions/admin-products-write`. Eksisterende `page_product_edit.jsx` er UI-stub uden backend; 3.1b hooker save/cancel-handlers op, tilføjer dirty-tracking + state-machine + inline banner-feedback, skifter form-data-source fra `window.MAD_DATA.PRODUCTS` til `useProducts()`, og co-locates write-adapter i `hooks.jsx` ved siden af eksisterende `remapProduct()` (read-adapter) for single source of mapping-truth.

3.1b er **edit-only**. Create + delete + publish-toggle = W5. Cloudinary upload = W4. Vægt/dimensioner = 3.1c (forward-compat via passthrough — ingen schema-ændring). Session-redirect ved 401 = separat MUST-før-launch issue (ikke 3.1b-scope).

## Scope

### In-scope (3.1b)

- Hook save/cancel-handlers op til 3.1a-endpoint
- Tilføj write-adapter `formToFields(form)` i `hooks.jsx` (co-located med `remapProduct`)
- Tilføj reverse kategori-mapping `CATEGORY_DISPLAY` i `hooks.jsx`
- Skift form-data-source fra `window.MAD_DATA.PRODUCTS` til `useProducts()` hook
- Component state-machine: idle → dirty → saving → saved/error
- Dirty-tracking via shallow JSON-equality (Save disabled ved clean form)
- Inline banner-feedback (success grøn, error rød) — fade efter 5s ved success
- Per-felt rendering af `validation_failed.details.fields.{key}: msg` (rød tekst under input)
- Cancel-confirm via native `window.confirm()` hvis dirty

### Out-of-scope

- Create new product — W5
- Delete product — W5
- Publish-toggle som dedicated workflow — W5 (men `visible→published` virker via partial-update i 3.1b)
- Cloudinary image upload — W4 (galleri-grid forbliver placeholder)
- Vægt + dimensioner felter — 3.1c (forward-compat: passthrough virker via 3.1a)
- Field-deletion via partial-update — post-launch (3.1a-kontrakt §)
- Multi-admin concurrency-håndtering — post-launch v1.1
- Optimistisk UI med rollback — pessimistisk + spinner valgt (Phase 2 #4)
- Toast-komponent — banner er nok til 3.1b-scope
- Modal-styled cancel-confirm — native `confirm()` valgt (Phase 2 #6)
- Session-redirect ved 401 — separat MUST-før-launch issue
- Internal-files-upload — W4 hooker upload (read-only display i 3.1b)
- Pre-POST client-validering — backend er sandhedskilde

## Integration: form ↔ 3.1a endpoint

Ingen endpoint-ændringer. Form rammer eksisterende 3.1a-endpoint per kontrakt:

```http
POST /.netlify/functions/admin-products-write
Content-Type: application/json
Cookie: nf_jwt=<...>     (eller Authorization: Bearer)

{
  "slug": "3d-printet-vase",
  "sku": "MAD-VASE-001",
  "fields": { "title": "...", "price": 79, "category": "Voksne", ... }
}
```

Browser bruger native `fetch` med `credentials: 'include'` så Identity-cookien medsendes (matcher `useStats`-pattern i `hooks.jsx:57-60`).

### Slug-source (faktum, verificeret 2026-04-26)

- `update-products.py:24,51` — `slug = fname[:-3]; data['slug'] = slug` (afledt fra filename uden `.md`)
- `products.json` indeholder `slug`-felt på hvert produkt
- `useProducts()` returnerer products via `remapProduct(p)` der bevarer `slug` (`hooks.jsx:24`)
- Form henter `slug` via `existing.slug` på det loaded product
- Slug er **IKKE editable** i 3.1b — påvirker filename på disk; ændring = delete+create (W5)

### SKU er identifier (3.1a-kontrakt)

- SKU er immutable, sendes som top-level sanity-check, ikke i `fields`
- Form viser SKU read-only (input markeres `disabled` eller `readOnly`)
- Adapter `formToFields` ekskluderer eksplicit `sku` fra output

## Adapter-placering: `hooks.jsx` (Option B — co-located read+write)

Eksisterende `remapProduct(p)` i `hooks.jsx:11-26` mapper products.json (kontrakt-format) → admin-v2 form-state. 3.1b tilføjer write-direction lige under den. **Single source of mapping-truth.**

Strukturen i `hooks.jsx` efter 3.1b-merge:

```js
// Read-direction (eksisterende)
const CATEGORY_SLUG = { 'Børn': 'born', 'Voksne': 'voksne', 'Erhverv': 'erhverv' };
function remapProduct(p) { /* products.json → form-state */ }

// Write-direction (NY i 3.1b)
const CATEGORY_DISPLAY = { 'born': 'Børn', 'voksne': 'Voksne', 'erhverv': 'Erhverv' };
function formToFields(form) { /* form-state → kontrakt-fields */ }
```

`formToFields` returnerer KUN editable fields. SKU + slug sendes top-level. `image`/`extra_images` håndteres ikke i 3.1b (W4).

## Component state-machine

| State | Triggers (event → next state) | UI-konsekvens |
|---|---|---|
| `idle` | initial / efter `saved`-fade | Save disabled, ingen banner |
| `dirty` | brugeren ændrer felt | Save enabled, ingen banner |
| `saving` | brugeren klikker Save | Save disabled + spinner, ingen banner |
| `saved` | 200 modtaget | Banner success, Save disabled, `originalForm` opdateres til ny saved value |
| `error` | 4xx/5xx modtaget | Banner error med detail, Save enabled (kan retry) |

**Auto-transitioner:**
- `saved` → `idle` efter 5s (setTimeout/useEffect, fader banner)
- `error` → `dirty` ved ny brugerændring (banner forbliver indtil ny save eller manual close)
- `error` → `saving` ved Save-klik

**Dirty-tracking:** shallow JSON-equality af `form` mod `originalForm`. **Caveat:** logikken antager primitive felter. Ved 3.1c (extra_images array) skal dirty-detect udvides til deep-equality. **Kode-kommentar i implementation skal nævne dette eksplicit.**

## Error-handling per HTTP-status

| HTTP | `error`-kode | UI-respons (DA / EN) | Notes |
|---|---|---|---|
| 400 | `invalid_request` | "Ugyldig forespørgsel — kontakt udvikler" / "Invalid request — contact developer" | Indikerer client-bug; skulle ikke ramme normalt |
| 400 | `validation_failed` | Per-felt fejl under inputs (rød tekst) + samlet banner: "Tjek felter markeret med fejl" / "Check fields marked with errors" | Brug `details.fields.{key}: msg` |
| 401 | `unauthorized` | Banner: "Du er logget ud — log ind igen" / "You're logged out — log in again" + manual login-link | **Interim** — overtages af MUST-før-launch session-redirect-issue når shippet |
| 403 | `forbidden` | Banner: "Du har ikke rettigheder til at gemme produkter" / "You don't have permission to save products" | Skulle ikke ramme normal admin |
| 404 | `product_not_found` | Banner: "Produktet findes ikke længere" / "Product no longer exists" + "Tilbage til liste"-knap | Sjælden — kun hvis produktet slettes mid-edit |
| 409 | `sku_mismatch` | Banner: "SKU-mismatch — genindlæs siden" / "SKU mismatch — reload page" | Frontmatter ændret out-of-band |
| 409 | `concurrent_write` | Banner: "Konflikt — en anden ændring kom først. Genindlæs siden." / "Conflict — another change won. Reload page." | Solo-admin: ~aldrig |
| 429 | `rate_limited` | Banner: "API rate-limit ramt — prøv igen om få minutter" / "API rate limit hit — try again in a few minutes" | 5000/time PAT-budget |
| 500 | `internal_error` | Banner: "Uventet fejl — prøv igen eller kontakt udvikler" / "Unexpected error — try again or contact developer" | Bug-indikator |
| 502 | `github_error` | Banner: "GitHub API-fejl — prøv igen om lidt" / "GitHub API error — try again in a moment" | Transitorisk |

i18n-nøgler for banner-tekster tilføjes til `admin-v2/src/i18n.jsx` i Phase 4.

## Form-data-flow

### Load (read-direction)

1. Bruger klikker produkt-række i `page_products.jsx` → `navigate('edit_product', { sku })`
2. `page_product_edit.jsx` mounts med `params.sku`
3. `useProducts()` returnerer products (allerede mapped via `remapProduct` til form-state-format)
4. `existing = products.find(p => p.sku === params.sku)`
5. Form initieres fra `existing` (vha. `React.useEffect` der reagerer når `useProducts()` skifter fra loading→loaded)
6. `originalForm` lagres til dirty-tracking
7. `slug = existing.slug` lagres til save-payload

### Save (write-direction)

1. Bruger klikker Save (kun enabled hvis dirty)
2. State → `saving`. Save disabled + spinner
3. `fields = formToFields(form)` mapper form → kontrakt-felter
4. `fetch('/.netlify/functions/admin-products-write', { method: 'POST', credentials: 'include', headers: {'Content-Type':'application/json'}, body: JSON.stringify({slug, sku, fields}) })`
5. På 200: state → `saved`, banner vises, `originalForm = form`, fade efter 5s → `idle`
6. På 4xx/5xx: state → `error`, banner med fejl-tekst (parse `details.fields` ved validation_failed), Save enabled

### Cancel

1. Bruger klikker Cancel
2. Hvis dirty: `if (!window.confirm(t('discard_changes_confirm'))) return;`
3. `navigate('products')`

## Banner UX

- **Success-banner:** grøn baggrund, tekst `t('saved_message')` = "Gemt — kunder ser ændringen om ca. 1 minut" / "Saved — customers will see the change in ~1 minute" (matcher `build_eta_seconds=60` fra 3.1a-kontrakt). Fader ud efter 5s.
- **Error-banner:** rød baggrund, fejl-tekst per HTTP-status (se tabel). **Fader IKKE automatisk** — brugeren skal læse + agere. Manual dismiss-knap (×) tilføjes; alternativt dismiss ved ny save-attempt.
- **Per-felt validation_failed:** rød tekst (small) under input-feltet. CSS-klasse `field-error` (eller lignende, beslut i Phase 4 stylesheet).

Genbrug-pattern: `page_products.jsx:60-68` viser inline error i tabel-card med "Prøv igen"-knap. 3.1b's banner følger samme stil.

## Auth-mekanisme (genbrug fra 3.1a-test)

Browser-fetch fra admin-v2 sender automatisk Identity-cookie ved `credentials: 'include'`. Memory siger cookie kan være `nf_jwt` eller `_nf-auth` (sidstnævnte HttpOnly). Fetch-kald respekterer begge — vi behøver ikke parse JWT'en selv.

Ved Phase 4-debug: hvis 401 ramt fejlagtigt, åbn DevTools Network-tab og verificér at `Cookie:`-header sendes på POST-requesten.
